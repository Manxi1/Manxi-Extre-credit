Activity Selection
greedy / 01

Given activities with start and finish times, select the maximum number of non-overlapping activities. The greedy rule is simple: always pick the activity that finishes earliest.

⏱ O(n log n) ◻ O(n) Interval Scheduling
Greedy Choice Always select the activity with the earliest finish time — this leaves the most remaining time for future activities. Proof: exchange argument shows any deviation cannot improve the count.
activity_selection.py 
def activity_selection(activities):
    # Sort by finish time — the greedy key
    sorted_acts = sorted(activities, key=lambda x: x[1])

    selected    = [sorted_acts[0]]
    last_finish = sorted_acts[0][1]

    for start, finish in sorted_acts[1:]:
        if start >= last_finish:   # no overlap
            selected.append((start, finish))
            last_finish = finish
    return selected

Huffman Encoding
greedy / 02

Construct an optimal prefix-free binary code for a set of symbols based on their frequencies. High-frequency symbols get shorter codes; rare symbols get longer ones.

⏱ O(n log n) ◻ O(n) Lossless Compression
Greedy Choice Always merge the two nodes with the lowest frequency. This builds a tree where frequently-used symbols naturally rise to the top (shorter codes).
huffman.py 
def build_huffman_tree(freq_table):
    heap = [HuffNode(freq=f, symbol=s)
            for s, f in freq_table.items()]
    heapq.heapify(heap)

    while len(heap) > 1:
        lo = heapq.heappop(heap)   # smallest
        hi = heapq.heappop(heap)   # 2nd smallest
        merged = HuffNode(
            freq=lo.freq + hi.freq,
            left=lo, right=hi)
        heapq.heappush(heap, merged)
    return heap[0]

Merge Sort
d&c / 01

Recursively split the array in half, sort each half, then merge the sorted halves. Guarantees O(n log n) in all cases — no worst-case degradation.

⏱ O(n log n) ◻ O(n) Stable Sort
Recurrence: T(n) = 2T(n/2) + O(n) Divide into 2 halves — T(n/2) each. Combine by merging — O(n). Master Theorem Case 2 gives Θ(n log n).
merge_sort.py 
def merge_sort(arr):
    if len(arr) <= 1:
        return arr          # base case

    mid   = len(arr) // 2
    left  = merge_sort(arr[:mid])   # conquer
    right = merge_sort(arr[mid:])   # conquer

    return _merge(left, right)     # combine

Binary Search
d&c / 02

Search a sorted array in O(log n) by repeatedly halving the search space. At each step, compare the midpoint and recurse on only one half.

⏱ O(log n) ◻ O(1) Sorted Array Search
Recurrence: T(n) = T(n/2) + O(1) Unlike Merge Sort, only ONE subproblem is pursued. Master Theorem gives Θ(log n) — with 1B elements, only 30 steps!
binary_search.py 
def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1
    while lo <= hi:
        mid = (lo + hi) // 2          # divide
        if   arr[mid] == target: return mid
        elif arr[mid] <  target: lo = mid + 1  # right
        else:                    hi = mid - 1  # left
    return -1

0/1 Knapsack Problem
dp / 01

Choose items (each used at most once) to maximize total value without exceeding weight capacity W. Greedy fails — DP builds a table of optimal sub-solutions bottom-up.

⏱ O(n × W) ◻ O(n × W) Combinatorial Optimization
DP Recurrence dp[i][w] = max( dp[i-1][w], dp[i-1][w-wᵢ] + vᵢ ) — for each item, either skip it or take it if it fits. Optimal substructure + overlapping subproblems → DP wins.
knapsack.py 
for i in range(1, n + 1):
    w_i, v_i = weights[i-1], values[i-1]
    for w in range(capacity + 1):
        dp[i][w] = dp[i-1][w]          # skip item
        if w_i <= w:
            dp[i][w] = max(
                dp[i][w],
                dp[i-1][w - w_i] + v_i  # take item
            )

Longest Common Subsequence
dp / 02

Find the longest sequence of characters appearing in both strings X and Y in the same order (not necessarily contiguous). Foundation of git diff and DNA alignment.

⏱ O(m × n) ◻ O(m × n) String Algorithms
DP Recurrence If X[i]==Y[j]: dp[i][j] = dp[i-1][j-1] + 1 Else: dp[i][j] = max(dp[i-1][j], dp[i][j-1]) Each cell reuses two previously computed sub-results.
lcs.py 
for i in range(1, m + 1):
    for j in range(1, n + 1):
        if X[i-1] == Y[j-1]:
            dp[i][j] = dp[i-1][j-1] + 1    # match
        else:
            dp[i][j] = max(
                dp[i-1][j],    # skip X[i]
                dp[i][j-1]     # skip Y[j]
            )

Traveling Salesman Problem
np / 01

Find the shortest tour visiting all n cities exactly once. NP-hard — no known polynomial algorithm. This implementation compares three approaches: brute force, exact Held-Karp DP, and a greedy heuristic.

⏱ O(n² · 2ⁿ) exact ◻ O(n · 2ⁿ) NP-Hard
Why NP-Hard? TSP is NP-complete in its decision form ("is there a tour ≤ k?"). The optimization version is NP-hard. For n=20: brute force needs 121 quadrillion checks; Held-Karp DP needs only ~419 million.

# dp[mask][v] = min cost reaching v, visiting cities in mask
dp[1][0] = 0.0   # start at city 0

for mask in range(1, 1 << n):
    for u in range(n):
        if not (mask & (1 << u)): continue
        for v in range(n):
            if mask & (1 << v): continue   # visited
            new_mask = mask | (1 << v)
            dp[new_mask][v] = min(
                dp[new_mask][v],
                dp[mask][u] + dist[u][v]
            )
